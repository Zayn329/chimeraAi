# Revision Guide: Comparative Analysis of Page Replacement Algorithms ## 1. Introduction to Page Replacement in Virtual Memory In modern operating systems, **Virtual Memory** allows processes to execute even when their memory requirements exceed the total available physical memory. When a process requests a page not currently in physical memory (a page fault), the system must bring that page in from disk. If all physical frames are occupied—a common occurrence under **heavy memory load**—the operating system must select a "victim" page to evict. This process is governed by **Page Replacement Algorithms**. ## 2. First-In-First-Out (FIFO) Algorithm The FIFO algorithm is the simplest form of page replacement. It operates on the principle of a queue, where the oldest page in physical memory is the first one selected for replacement. ### Characteristics: * **Implementation:** It is "easy to program" as it only requires a pointer or a queue to track the order of page arrival. * **Performance:** Under heavy load, FIFO often performs poorly. In the provided context, FIFO resulted in **fifteen page faults** compared to LRU’s twelve. * **Belady’s Anomaly:** A significant drawback of FIFO is that it "suffers from Belady's anomaly," a phenomenon where increasing the number of available frames can actually result in an *increase* in the number of page faults. ## 3. Least Recently Used (LRU) Algorithm The LRU algorithm is based on the **Temporal Locality** principle: pages used recently are likely to be used again soon. It replaces the page that has not been used for the longest period of time. ### Characteristics: * **Performance:** LRU is considered an "approximation of optimal page replacement". In comparative tests, LRU produced **twelve faults**, making it "much better than FIFO replacement". * **Implementation Challenges:** While effective, LRU is "difficult to implement" and may require "substantial hardware assistance" (such as counters or a stack) to track the time of last use for every page. * **Logic:** When a fault occurs, LRU identifies the page used least recently. For example, if page 2 was used least recently among the frames, it is replaced, even if it might be needed shortly thereafter. ## 4. Comparative Summary under Heavy Memory Load | Feature | FIFO | LRU | | :--- | :--- | :--- | | **Logic** | Replaces the oldest page. | Replaces the page unused for the longest time. | | **Efficiency** | Generally lower; prone to high fault rates. | Higher; "considered to be good". | | **Complexity** | Low; easy to program. | High; requires hardware support. | | **Anomaly** | Suffers from Belady's Anomaly. | Does not suffer from Belady's Anomaly. | | **Fault Count** | Higher (e.g., 15 faults). | Lower (e.g., 12 faults). | ## 5. Conclusion Under heavy memory load, the choice of algorithm significantly impacts system throughput. While **FIFO** is computationally inexpensive, its lack of regard for page usage patterns leads to inefficiencies like Belady's Anomaly. Conversely, **LRU** provides a more sophisticated approximation of the **Optimal Page Replacement** strategy by leveraging historical usage data, resulting in fewer page faults and better utilization of virtual memory. Most modern systems use approximations of LRU, such as the **Second-Chance Algorithm**, to balance performance with implementation costs. *** **References:** * *Operating System Concepts - Question Bank Topics (Sections 15, 16, 17, 19).* * *Contextual Analysis of FIFO vs. LRU Fault Metrics (15 vs. 12 faults).*
